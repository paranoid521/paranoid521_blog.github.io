git使用简介

工作区–>暂存区–>本地库—–》远程库

当修改了某个文件，给文件的修改是在工作区，
git add . 则是把工作区的修改放到暂存区，每当有修改，运行git add .都会把修改放到到暂存区
git commit -m “内容” git commit则是把暂存区所有git add的修改都提交到本地代码库。双引号里的内容为说明修改了哪些内容
git push origin master 把本地代码库推送到远程代码库，远程库的默认名是origin
